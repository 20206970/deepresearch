# LangGraph Deep Research 项目技术文档

## 项目概述

基于 LangGraph 的深度研究 Agent 系统，通过多 Agent 协作完成复杂主题的深度研究任务。

### 核心能力
- 自动分解研究主题为多个子任务
- 并行执行搜索与信息汇总
- 生成结构化研究报告（包含来源引用）
- 长期记忆复用，避免重复研究

---

## 一、架构设计

### 1.1 整体架构

```
用户请求 → Planner Agent → 并行任务执行 → Reporter Agent → 最终报告
              ↓                                    ↑
         短期记忆 + 长期记忆 ←————————————→ 存储新记忆
```

### 1.2 技术栈
- **图编排**: LangGraph (StateGraph)
- **LLM**: OpenAI API (支持自定义 base_url)
- **搜索**: Tavily API
- **短期记忆**: ConversationSummaryBufferMemory
- **长期记忆**: ChromaDB (向量数据库)
- **前端**: Vue 3 + Vite
- **后端**: FastAPI

## 二、记忆管理系统（重点）

### 2.1 双层记忆架构

```
┌─────────────────────────────────────────────────────────┐
│                    短期记忆 (Session级)                  │
│  ConversationSummaryBufferMemory                        │
│  - 保存最近对话                                         │
│  - token 超过阈值自动摘要                                │
│  - 当前会话上下文                                        │
└─────────────────────────────────────────────────────────┘
                          ↓ 上下文提取
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    长期记忆 (持久化)                      │
│  ChromaDB + VectorStoreRetrieverMemory                  │
│  - 向量语义检索                                          │
│  - 跨会话历史研究                                        │
│  - 主题相关性匹配                                        │
└─────────────────────────────────────────────────────────┘
```

### 2.2 短期记忆实现

**文件**: `src/memory/short_term.py`

```python
# 核心实现
ConversationSummaryBufferMemory(
    llm=llm,
    max_token_limit=2000,  # 超过阈值自动生成摘要
    return_messages=True,
)
```

**工作原理**:
1. 保存用户输入和 Agent 输出到内存
2. 当 token 数超过 `max_token_limit` 时，使用 LLM 生成对话摘要
3. 释放旧的具体消息，保留摘要
4. `get_memory_context()` 返回格式化的最近 4 条对话

**面试提问准备**:
- **Q: 为什么选择 ConversationSummaryBufferMemory?**
  - A: 结合了"对话缓冲"和"自动摘要"两种能力。当对话较短时保留完整历史，当对话较长时自动压缩，避免上下文溢出。

- **Q: 如何控制内存大小?**
  - A: 通过 `max_token_limit` 参数，默认 2000 tokens。LangChain 内部会监控 token 使用情况，超限时触发摘要生成。

### 2.3 长期记忆实现

**文件**: `src/memory/long_term.py`

```python
# 核心实现
VectorStoreRetrieverMemory(
    retriever=Chroma(...).as_retriever(search_kwargs={"k": 3}),
    memory_key="chat_history",
)
```

**工作原理**:
1. 使用 ChromaDB 向量存储持久化记忆
2. 研究完成后，将主题、任务结果、报告整体存入向量库
3. 下次研究相似主题时，通过语义检索获取相关记忆
4. 检索返回 top-K (默认 3 条) 最相关的历史研究

**数据流**:
```
研究报告 → ChromaDB (向量存储)
     ↓
下次研究 "相关主题" → 语义检索 → 返回历史研究摘要
```

**面试提问准备**:
- **Q: 为什么需要两层记忆?**
  - A: 短期记忆处理当前会话的即时上下文，长期记忆实现跨会话的知识积累。短期记忆的摘要可以作为长期记忆的输入，长期记忆的检索结果可以作为短期记忆的补充。

- **Q: 长期记忆如何实现语义检索?**
  - A: 使用向量嵌入 + ChromaDB。当存入记忆时，将内容转换为向量；当检索时，将查询转换为向量，通过余弦相似度找到最相似的记忆。

## 三、任务编排与执行流程

### 3.1 LangGraph 工作流

**文件**: `src/graph/research.py`

```python
workflow = StateGraph(ResearchState)

workflow.add_node("planner", planner_node)           # 任务规划
workflow.add_node("search_summarize", search_summarize_node)  # 并行搜索汇总
workflow.add_node("reporter", reporter_node)         # 报告生成

workflow.add_edge(START, "planner")
workflow.add_conditional_edges("planner", _split_tasks, ["search_summarize"])
workflow.add_edge("search_summarize", "reporter")
workflow.add_edge("reporter", END)
```

### 3.2 执行流程

```
1. 用户输入主题
   ↓
2. Planner Node
   - 读取短期记忆 + 长期记忆作为上下文
   - 调用 LLM 生成任务列表 (JSON格式)
   - 返回 tasks: [{id, title, intent, query}, ...]
   ↓
3. 任务分发 (_split_tasks)
   - 将任务列表拆分为并行分支
   - 每个任务独立执行 search_summarize_node
   ↓
4. Search/Summarize Node (并行)
   - 调用 search_web 工具获取搜索结果
   - 调用 LLM 生成任务总结
   - 提取来源 URL
   - 返回 task_results + sources
   ↓
5. Reporter Node
   - 汇总所有任务结果
   - 传入参考来源链接
   - 调用 LLM 生成最终报告
   - 保存到长期记忆
   ↓
6. 返回最终报告
```

### 3.3 并行执行机制

使用 LangGraph 的 `Send` 实现任务并行：

```python
def _split_tasks(state: dict) -> list[Send]:
    tasks = state.get("tasks", [])
    return [Send("search_summarize", {**state, "task": task}) for task in tasks]
```

**面试提问准备**:
- **Q: LangGraph 的条件边和普通边有什么区别?**
  - A: 普通边是有向无环的固定流向，条件边可以根据状态动态决定下一个节点。`_split_tasks` 返回一个节点列表，实现一对多的并行分发。

- **Q: 并行任务如何保证状态正确汇总?**
  - A: 使用 `Annotated[list, operator.add]` 标记字段。LangGraph 会自动将各并行节点的输出合并到同一个列表中。

## 四、Agent 设计

### 4.1 三个核心 Agent

| Agent | 角色 | 工具 | 职责 |
|-------|------|------|------|
| Planner | 任务规划 | 无 | 分析主题，生成研究任务列表 |
| Summarizer | 搜索执行 | search_web, create_note, read_note, update_note | 执行搜索，生成任务总结 |
| Reporter | 报告撰写 | create_note, read_note | 汇总结果，生成最终报告 |

### 4.2 ReAct 模式

使用 LangGraph 内置的 `create_react_agent`：

```python
create_react_agent(
    llm,
    tools=[search_web, create_note, ...],
    prompt=SYSTEM_PROMPT,
)
```

ReAct (Reasoning + Acting) 模式让 Agent 可以：
1. 思考下一步做什么
2. 选择并调用工具
3. 观察工具结果
4. 决定是否继续

## 五、状态管理

### 5.1 ResearchState 定义

**文件**: `src/state.py`

```python
class ResearchState(dict):
    topic: str = ""                    # 研究主题
    tasks: list = []                   # 任务列表
    task_results: Annotated[list, operator.add] = []  # 并行任务结果
    sources: Annotated[list, operator.add] = []        # 来源列表
    report: Optional[str] = None       # 最终报告
    loop_count: int = 0                # 循环计数
    memory_context: str = ""           # 记忆上下文
```

### 5.2 Sources 结构

```python
class SearchSource(BaseModel):
    query: str = ""     # 搜索查询
    url: str = ""       # 来源 URL
    title: str = ""     # 来源标题
```

**面试提问准备**:
- **Q: 为什么不把所有信息存在 ChromaDB，还要笔记功能?**
  - A: 笔记是 Agent 主动记录的结构化信息，长期记忆是自动存入的研究结果。笔记更像 Agent 的"工作台"，可以临时存放中间结果。

## 八、项目亮点总结

### 面试话术

> "我实现了一个基于 LangGraph 的深度研究 Agent 系统，主要特点包括：
>
> 1. **双层记忆系统**：短期记忆用 ConversationSummaryBufferMemory 处理会话上下文，长期记忆用 ChromaDB 向量库实现跨会话知识复用
> 2. **并行任务执行**：通过 LangGraph 的条件边和 Send 实现多任务并行，提升研究效率
> 3. **多 Agent 协作**：Planner 负责任务规划，Summarizer 执行搜索和总结，Reporter 生成最终报告
> 4. **来源追溯**：搜索结果中的 URL 会传递到报告生成环节，最终报告中包含可点击的参考来源链接
> 5. **工具增强**：Agent 可调用搜索和笔记工具，扩展了纯对话能力"

### 技术难点

- **记忆的生命周期管理**：短期→长期的转换逻辑
- **并行状态合并**：使用 `operator.add` 实现多节点结果聚合
- **来源信息传递**：JSON 解析 + 属性/字典兼容访问
