# LangGraph 科研助手前端设计

## 项目概述

- **项目名称**: LangGraph Deep Research Frontend
- **类型**: 单页 Web 应用 (Vue 3 + Vite)
- **核心功能**: 配合后端 LangGraph 工作流，提供分步式研究向导界面
- **目标用户**: 研究人员、开发者、需要进行深度调研的用户

## UI/UX 设计

### 布局结构

```
┌─────────────────────────────────────────────────────────┐
│  Header: Logo + 标题                                    │
├─────────────────────────────────────────────────────────┤
│  Progress Bar: 步骤进度指示器                           │
```

### 响应式断点

- Mobile: < 640px (单列布局)
- Tablet: 640px - 1024px (居中容器)
- Desktop: > 1024px (最大宽度 900px 居中)

**配色方案**

- 背景次色: #1e293b (稍浅)
- 主强调色: #3b82f6 (亮蓝)
- 成功色: #22c55e (绿色)
- 文字主色: #f1f5f9
- 文字次色: #94a3b8
- 边框色: #334155

**字体**
- 标题: "Noto Serif SC", serif (学术感)
- 正文: "Inter", system-ui, sans-serif
- 代码/报告: "JetBrains Mono", monospace

**间距系统**
- 基础单位: 4px
- 容器内边距: 24px
- 卡片间距: 16px
- 元素间距: 12px

**视觉效果**
- 卡片: 半透明背景 (bg-opacity-80%) + backdrop-blur

- 圆角: 12px (卡片), 8px (按钮), 4px (输入框)

### 组件设计

**1. 进度指示器**
- 4 个圆点表示 4 步骤
- 当前步骤: 亮蓝色填充 + 脉冲动画
- 已完成: 绿色填充 + ✓ 图标
- 未完成: 灰色边框
- 步骤之间有连线

**2. 主题输入卡片**
- 大型输入框 (textarea)
- 占位文字: "输入您想研究的主题..."
- "开始研究" 按钮 (主色调，hover 发光)

**3. 任务列表卡片**
- 每个任务一个卡片
- 左侧: 状态图标 (⏳ 进行中 / ✓ 完成)
- 标题: 任务名称 (加粗)
- 描述: 任务意图 (次要颜色)
- 底部: 检索关键词 (标签样式)

**4. 研究结果卡片**
- 每个任务一个折叠面板
- 展开显示该任务的研究总结
- Markdown 渲染

**5. 报告卡片**
- 全屏 Markdown 渲染
- 代码块高亮
- 表格样式美化

**6. 操作按钮**
- 主按钮: 亮蓝背景，白色文字
- 次按钮: 透明背景，边框
- 加载状态: 按钮内显示 spinner

## 功能规范

### 步骤 1: 主题输入

- 输入研究主题 (必填，最少 2 字符)
- 点击"开始研究"调用后端 `/research`
- 进入下一步，显示 Planner 生成的任务列表
- 错误提示: Toast 通知

### 步骤 2: 任务规划

- 显示 Planner 生成的任务列表 (通常 3-7 个)
- 每个任务显示: 标题、意图、检索关键词
- 点击"执行任务"进入下一步

### 步骤 3: 研究执行

- 显示任务列表，每个任务显示状态
- 所有任务完成后显示汇总
- 点击"生成报告"进入下一步

### 步骤 4: 报告生成

- 显示最终 Markdown 报告
- 支持 Markdown 渲染 (标题、列表、代码块、表格)
- 点击"新建研究"返回步骤 1

### 边界情况

- 网络错误: 显示重试按钮
- 后端超时: 60s 后提示失败，可重试
- 空任务列表: 显示默认任务
- 报告生成失败: 显示错误信息 + 重试

## 技术栈

- Vue 3 (Composition API)
- Vite (构建工具)
- marked (Markdown 解析)
- highlight.js (代码高亮)
- axios (HTTP 请求)
